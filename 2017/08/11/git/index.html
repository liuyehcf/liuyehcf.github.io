13 .gitignore

基础规则

  1. 空白行，不匹配任何文件，仅增加可读性
  2. 规则以#开头，表示注释
  3. 规则行尾的空格，会被忽略，除非用\进行转义
  4. 规则以!开头，表示反转该规则（.gitignore文件默认的语义是忽略，反转后表示包含）
  5. 规则以/结尾，表示匹配目录以及该目录下的一切
  6. 规则以不包含/，表示匹配文件或目录
  7. *匹配多个字符，不包括/
  8. ?匹配单个字符，不包括/
  9. []匹配指定的多个字符
  10. 规则以/开头，表示以根目录开始，即匹配绝对路径。例如/*.c匹配cat-file.c，但不匹配mozilla-sha1/sha1.c
  11. **/开头，表示匹配所有目录。例如**/foo匹配foo目录或文件。**/foo与foo的作用是一样的
  12. /**开头，表示匹配内部的一切。例如abc/**匹配abc目录下的所有文件。abc/**与abc/的作用是一样的
  13. /**/表示匹配0个或多个目录。例如a/**/b匹配a/b、a/x/b、a/x/y/b
