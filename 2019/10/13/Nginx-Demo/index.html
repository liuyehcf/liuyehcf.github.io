1 匹配规则

优先级从上到下

  1. location =：精确匹配
  2. location <absolute path>：完整路径
  3. location ^~ <path>：以某个字符串开头，非正则
  4. location ~|~* <regex>：正则匹配，~表示大小写敏感，~*表示大小写不敏感
  5. location <relative path>：相对路径
  6. location /：通用匹配
